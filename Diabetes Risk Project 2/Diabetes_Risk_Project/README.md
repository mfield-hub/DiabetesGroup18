# Diabetes Risk Prediction & Lifestyle Segmentation

A machine learning web application that predicts diabetes risk stage and lifestyle cluster from patient data.

## Project Structure

```
Diabetes_Risk_Project/
├── data/
│   └── Diabetes_and_LifeStyle_Dataset_.csv
├── models/                        # Saved .pkl model files (generated by running src scripts)
│   ├── dt_model.pkl
│   ├── rf_model.pkl
│   ├── xgb_model.pkl
│   ├── kmeans_model.pkl
│   ├── kmeans_scaler.pkl
│   ├── label_encoder.pkl
│   └── feature_columns.pkl
├── notebooks/
│   ├── Data_Understanding.ipynb
│   └── Data_Preparation.ipynb
├── src/
│   ├── data_preparation.py        # Load, clean, encode, split data
│   ├── model_training.py          # Train DT, RF, XGBoost classifiers
│   └── clustering.py              # K-Means lifestyle clustering
├── app.py                         # Dash web application
├── requirements.txt
└── README.md
```

## Models

| Model | Type | Target |
|---|---|---|
| Decision Tree | Classification | diabetes_stage |
| Random Forest | Classification | diabetes_stage |
| XGBoost | Classification | diabetes_stage |
| K-Means (k=3) | Clustering | lifestyle_group |

**Target classes:** No Diabetes, Pre-Diabetes, Type 1, Type 2, Gestational

**Lifestyle cluster features:** BMI, physical activity, sleep hours, alcohol consumption, screen time

## Setup & Run

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Train models (first time only)
```bash
cd src
python model_training.py
python clustering.py
```

### 3. Run the web app
```bash
python app.py
```
Then open http://localhost:8050 in your browser.

## Deployment (Render)

1. Push this project to a GitHub repository
2. Create a new **Web Service** on [render.com](https://render.com)
3. Set:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:server`
4. Add environment variable if needed: `PYTHON_VERSION=3.11`
